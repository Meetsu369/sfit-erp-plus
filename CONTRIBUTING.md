# Contributing

Thank you for your interest in contributing to SFIT ERP+.

## Getting Started

1. Fork the repository.
2. Create a feature branch.
3. Make your changes.
4. Test the application.
5. Submit a Pull Request.

## Branch Naming

feature/login

feature/attendance

feature/timetable

feature/profile

bugfix/login

hotfix/session

## Commit Messages

Use meaningful commit messages.

Examples:

feat: add attendance module

fix: resolve login session issue

docs: update README

refactor: improve router structure

## Coding Guidelines

- Follow Flutter best practices.
- Use Material 3 components.
- Keep widgets reusable.
- Keep business logic separate from UI.
